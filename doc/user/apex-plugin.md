# EspreSQL — APEX Region Plug-in Setup Guide

This guide explains how to install EspreSQL as an Oracle APEX **Region Plug-in** so that it can be added to any page via drag-and-drop in the Page Designer.

---

## How it works

The plug-in renders EspreSQL directly into the APEX page (no iframe). The PL/SQL render function emits the full editor HTML and loads two CSS files plus two JS files from the static files you upload. The result is a self-contained EspreSQL editor that coexists with the APEX page layout.

---

## Prerequisites

| Item | Version |
|---|---|
| Oracle APEX | 22.1 or later |
| EspreSQL static files | built (`npm run build`, then upload) |
| APEX parsing schema | must own the `espresql_plugin` package |

---

## Step 1 — Build and upload static files

Run the full build first so all output files are generated:

```bash
npm run build
```

Upload the following files to **Shared Components → Static Application Files**, inside a versioned subfolder (e.g. `espresql/v2/`), preserving the subfolder structure shown:

| File | From | Target path in APEX |
|---|---|---|
| `web/app.css` | `web/` | `espresql/v2/web/app.css` |
| `web/app_all.js` | `web/` (built) | `espresql/v2/web/app_all.js` |
| `web/highlight.js` | `web/` | `espresql/v2/web/highlight.js` |
| `web/settings.js` | `web/` | `espresql/v2/web/settings.js` |
| `web/tabs.js` | `web/` | `espresql/v2/web/tabs.js` |
| `web/autocomplete.js` | `web/` | `espresql/v2/web/autocomplete.js` |
| `web/erd.js` | `web/` | `espresql/v2/web/erd.js` |
| `web/state.js` | `web/` | `espresql/v2/web/state.js` |
| `dist/antv-x6.min.js` | `dist/` | `espresql/v2/dist/antv-x6.min.js` |
| `apex/apex-plugin.css` | `apex/` | `espresql/v2/apex/apex-plugin.css` |

> **Note:** `web/app_all.js` is the single-file bundle that replaces all individual `web/*.js` imports. It is generated by `npm run build:web` (included in `npm run build`). The individual `web/*.js` files above are only needed if you want to use the standalone `index.html` — the plug-in itself only needs `web/app.css`, `web/app_all.js`, `dist/antv-x6.min.js`, and `apex/apex-plugin.css`.

After upload, the base URL for the files is `#APP_IMAGES#espresql/v2/` (substitute your folder name).

---

## Step 2 — Install the render package

Run the following script as the APEX workspace parsing schema:

```sql
@@apex/create_plugin.sql
```

This compiles the `espresql_plugin` PL/SQL package (spec + body). Verify with:

```sql
select object_name, object_type, status
from   user_objects
where  object_name = 'ESPRESQL_PLUGIN';
```

Both rows must show `VALID`.

---

## Step 3 — Create the plug-in definition

In APEX Builder: **Shared Components → Other Components → Plug-ins → Create**.

| Field | Value |
|---|---|
| Name | `EspreSQL` |
| Internal Name | `IT.RADICLE.ESPRESQL` |
| Type | **Region** |
| Category | Content |
| Render Function Name | `espresql_plugin.render` |
| Version | `2.0.0` |

### Add plug-in attributes

Go to the **Attributes** tab and add these four component-level attributes:

| # | Label | Type | Required | Default |
|---|---|---|---|---|
| 1 | Theme | Select List | No | `apex` |
| 2 | Height | Text | No | `calc(100vh - 120px)` |
| 3 | Show Share Button | Checkbox | No | `N` (unchecked) |
| 4 | Static Files Base URL | Text | **Yes** | `#APP_IMAGES#espresql/` |

For attribute 1 (Theme), add these Static LOV entries:

| Display | Return |
|---|---|
| APEX (Ocean Dark) | `apex` |
| Dark | `dark` |
| Light | `light` |

For attribute 4 (Static Files Base URL), the help text should read:  
*"Path prefix for the uploaded EspreSQL static files, ending with `/`. Must preserve the `web/`, `dist/`, and `apex/` subfolder structure. Supports `#APP_IMAGES#` and `#WORKSPACE_IMAGES#` substitutions. Example: `#APP_IMAGES#espresql/v2/`"*

Click **Apply Changes** to save the plug-in.

---

## Step 4 — Add the region to a page

1. Open the target page in **Page Designer**.
2. In the **Regions** pane, click **+** → **Region**.
3. Set **Type** to **EspreSQL** (the new plug-in appears in the list).
4. Set **Title** to anything (or leave blank).
5. In the **Attributes** section configure:
   - **Theme**: `APEX (Ocean Dark)` (recommended for Redwood pages)
   - **Height**: `calc(100vh - 120px)` — adjust to subtract your page's header height
   - **Static Files Base URL**: the root subfolder where you uploaded the files (e.g. `#APP_IMAGES#espresql/v2/`)
6. Save and run.

### Recommended page settings

| Setting | Recommended value |
|---|---|
| Page template | Minimal / No Header (or Standard with nav suppressed) |
| Region template | Blank with Attributes |
| Page body overflow | `hidden` (avoids double scroll bar) |

## Troubleshooting

**Editor blank / unstyled**  
Check the browser network panel. The most common cause is a wrong *Static Files Base URL* — verify that `app.css` and `app_all.js` are reachable at `{base}web/app.css` and `{base}web/app_all.js`.

**APEX page layout broken after adding the plug-in**  
`apex-plugin.css` must be uploaded and the base URL correct. Its job is to reset the `body` flex layout that `app.css` sets. If it is missing, the APEX page body gets `display: flex; height: 100vh` which breaks the layout.

**Wrong theme / no theme toggle**  
The theme is driven by the `Theme` plug-in attribute. `apex` hides the toggle button (theme is fixed). Use `dark` or `light` to let users toggle.

**ER Diagram tab blank**  
`antv-x6.min.js` must be uploaded at `{base}dist/antv-x6.min.js` and must load before `app_all.js`. Both are loaded via `apex_javascript.add_library` in order — verify the file exists at the expected path.

**Package invalid after upgrade**  
Re-run `apex/create_plugin.sql` after any EspreSQL upgrade that changes the package interface. Also re-upload the static files.

## Files summary

```
apex/
  espresql_plugin.pks   — package spec  (render function signature)
  espresql_plugin.pkb   — package body  (render function implementation)
  apex-plugin.css        — CSS adapter: resets body flex, scopes plugin styles
  create_plugin.sql      — installs the package (run once per schema)
web/
  app.css               — EspreSQL UI styles (upload to APEX)
  app_all.js            — bundled app (npm run build:web → upload to APEX)
dist/
  antv-x6.min.js        — AntV X6 graph library (upload to APEX)
doc/user/
  apex-plugin.md        — this guide
  apex-embedding.md     — original iframe embedding guide (without plug-in)
```
