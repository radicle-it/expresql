                Quick Start
                

                    
Write table names at the left margin. Indent columns underneath. Indent a table under another to create a parent-child FK automatically.

                    departments
   name /nn
   location
   employees
      name /nn vc100
      email /lower
      salary num(10,2)
                    
Add settings at the end of the script:

                    # settings = { db: "23c", auditcols: yes, prefix: "app_" }
                    
The DDL panel updates in real time. Use Copy SQL to grab the output. Load a starter schema from the Examples ▾ button in the toolbar.

                

            

            

                Keyboard Shortcuts
                

                    

                        
TabIndent 3 spaces

                        
EnterAuto-indent to match current line

                        
EscClose Help and Settings panels

                    

                    
DDL updates in real time on every keystroke. The ER diagram updates 350 ms after the last keypress.

                    
QSQL text is auto-saved to localStorage and restored on next visit. Each schema tab has its own content.

                

            

        


        

            

                Tables
                

                    

                        
table_nameCreates a table with auto primary key

                        
name /insert NGenerate N sample data rows (max 1000)

                        
name /restEnable ORDS REST on the table

                        
name /auditAdd AUDIT ALL ON <table>

                        
name /auditcolsAdd CREATED, CREATED_BY, UPDATED, UPDATED_BY columns + trigger

                        
name /rowversionAdd ROW_VERSION column; trigger increments on every UPDATE (OCC)

                        
name /apiGenerate Table API (TAPI package)

                        
name /api /auditlog [log_table]Layered TAPI + audit package with PRAGMA AUTONOMOUS_TRANSACTION

                        
name /colprefix pfxPrefix all columns of this table with pfx

                        
name /compressCREATE TABLE … COMPRESS

                        
name /flashback archiveEnable Flashback Data Archive (optional archive name)

                        
name /immutableAppend-only table — trigger blocks UPDATE/DELETE (21c+)

                        
name /historyTemporal history table with valid_from/valid_to (23ai+)

                        
name /sodaSODA JSON document collection with fixed schema (21c+)

                    

                    
Tables indented under another table automatically become children with a FK to the parent.

                Annotations
                

                    
Oracle SQL annotations can be added to tables, columns, and views using curly braces {} after the name or directive list.

                    

                        
{Key 'value'}Key-value annotation — single or double quotes

                        
{FlagKey}Flag annotation (no value)

                        
{DESCRIPTION 'text'}Also generates COMMENT ON TABLE/COLUMN

                        
{GROUP 'name'}Creates an Oracle annotation group

                    

                    departments {DESCRIPTION 'HR departments', Classification 'HR'}
   name /nn {DESCRIPTION 'Department name'}
                

            

            

                Comments
                

                    
Comments are ignored by the DDL generator.

                    

                        
-- textSingle-line comment

                        
/* text */Multi-line comment

                        
[text]Inline comment (square brackets)

                    

                    -- My schema
employees   [main workforce table]
   name vc100 /* full legal name */

                Schema Migration
                

                    
Write the old schema, add a separator line starting with # and at least two dashes, then write the new schema. The DDL panel switches to migration mode automatically.

                    employees
   name vc100 /nn
   email vc200

# ---

employees
   name vc100 /nn
   email vc200 /nn
   phone vc50
                    
Use the ⇄ Compare button to insert the separator automatically.

                    

                        
No separatorNormal DDL mode — full CREATE TABLE script

                        
With separatorMigration mode — ALTER TABLE / CREATE TABLE incremental script

                        
⚠ DESTRUCTIVEColumn dropped, NOT NULL added — requires manual step

                        
⚠ LOSSYColumn size reduced or type changed — data may be truncated

                        
INFOSuspected column rename — verify before applying

                    

                

            

            

                Settings
                

                    
Use the ⚙ Settings panel to configure options visually, or add a # settings = { … } line to your script.

                    # settings = { pk: seq, semantics: CHAR, db: 23c }
                

            

        


        

            

                Quick Examples
                

                    
Schema migration — write the old schema, then # ---, then the new schema:

                    employees
   name vc100 /nn
   email vc200

# ---

employees
   name vc100 /nn
   email vc200 /nn
   phone vc50

departments
   dept_name vc100 /nn
                    
Schema with automatic parent-child hierarchy:

                    departments /insert 2
   name /nn
   employees /insert 5
      name /nn vc50
      email /lower
      salary num(10,2)
                    
Explicit FK + audit columns:

                    orders /audit /insert 10
   customer_id /fk customers
   total num(10,2) /nn
   status /check OPEN,CLOSED,PENDING
                    
Layered TAPI:

                    employees /api full+hks
   name vc100 /nn
   row_version num /nn
